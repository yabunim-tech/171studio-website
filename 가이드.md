# 171studio 웹사이트 작업 가이드

노코드 베이스로 Claude Code와 작업할 때 참고하는 문서. "이거 어떻게 하더라" 싶을 때 이 파일부터 찾아보면 됩니다.

> 이 문서는 계속 업데이트됩니다. 새로 중요한 결정이나 다시 볼 만한 내용이 생기면 Claude가 이 파일에 알아서 추가합니다 (프로젝트 `CLAUDE.md`에 그렇게 지시해뒀어요) — 사용자님이 따로 "가이드에 추가해줘"라고 안 하셔도 됩니다.

## 2. 미리보기 (로컬에서 먼저 확인)

배포 전에 내 컴퓨터에서만 보는 방법:

```
cd ~/Desktop/171studio-website
npm run dev
```

터미널에 "Ready" 뜨면 브라우저에서 `http://localhost:3000` 접속. 코드 고치고 저장하면 자동으로 새로고침됩니다.

---

## 3. 실제 사이트에 반영하기 (배포)

**Claude한테 요청하는 경우:** "이거 고쳐서 배포까지 해줘"라고 하면 커밋 → GitHub 업로드 → 자동배포까지 한 번에 처리합니다. 가장 편한 방법.

**직접 하고 싶은 경우** (VS Code 소스 제어 탭에서):
1. 수정한 파일들 옆 `+` 눌러서 스테이징
2. 커밋 메시지 아무거나 입력하고 체크 표시(✓) 눌러서 커밋
3. 위쪽 "동기화" 또는 "Push" 버튼 클릭

푸시되고 나면 자동으로 실제 사이트에 반영됩니다.

## 4. 자주 수정하는 파일 위치

`src` 폴더 안에서만 작업하면 됩니다. `node_modules`, `.next` 폴더는 라이브러리/빌드 캐시라 신경 안 써도 됩니다. VS Code에서 `Cmd+P` 누르고 파일명 일부만 쳐도 바로 찾아집니다.

| 화면/내용 | 파일 |
|---|---|
| 메인 홈 (히어로/추천상품/브랜드 밴드) | `src/app/page.tsx` |
| 소개 페이지 (브랜드 스토리/오시는 길) | `src/app/about/page.tsx` |
| 제품 목록 | `src/app/products/page.tsx` |
| 제품 상세 | `src/app/products/[slug]/page.tsx` |
| 장바구니 | `src/app/cart/page.tsx` |
| 상단 헤더 | `src/components/SiteHeader.tsx` |
| 하단 푸터 | `src/components/SiteFooter.tsx` |
| **매장 정보** (주소·전화·영업시간) | `src/lib/store-info.ts` — 여기 한 곳만 고치면 about 페이지·footer·지도에 전부 자동 반영 |
| 상품 데이터 (가격·이름·카테고리) | `src/lib/data/products.ts` |
| 로고 이미지 | `public/brand/171studio-logo.png` |

## 5. 품절 처리 직접 하기 (자주 할 일이라 VS Code에서 바로 해도 됨)

상품 재고 상태는 `src/lib/data/products.ts`에서 상품 하나당 이 두 값으로 관리합니다:

```ts
inStock: true,        // false로 바꾸면 "품절" 표시되고 담기 버튼 비활성화
stockQuantity: 24,     // 남은 수량. 굳이 정확할 필요 없으면 0으로만 바꿔도 됨
```

**품절로 바꾸려면** 해당 상품을 찾아서 (`Cmd+F`로 상품명 검색) 이렇게 바꾸면 끝:

```ts
inStock: false,
stockQuantity: 0,
```

예시로 8번째 상품("작은 책방 에나멜 뱃지")이 이미 이렇게 품절 처리되어 있어요 — 그거 보고 똑같이 따라 하면 됩니다.

**"품절임박"만 표시하고 싶을 때** (아직 구매는 가능, 배지만 노출)는 `inStock`은 `true`로 두고, `badges: []` 부분에 `"low_stock"`을 추가하면 됩니다: `badges: ["low_stock"]`

다 고치면 평소처럼 저장(Cmd+S) → 커밋 → push만 하면 1분 안에 실제 사이트에 반영돼요. 이건 값만 바꾸는 작업이라 문법 깨질 위험이 거의 없어서, Claude 없이 직접 하기 가장 좋은 작업 중 하나입니다.

## 6. 새 대화로 이어가기 (토큰 관리)

대화가 길어지면 새 대화창을 여는 게 나을 수 있어요. 새로 열 때 이렇게 시작하면 바로 상황 파악이 됩니다:

> "171studio 웹사이트(`~/Desktop/171studio-website`) 이어서 작업하려고 해. 프로젝트 안에 `가이드.md` 파일 있으니까 그거 먼저 읽고 현재 상황 파악해줘. [오늘 하고 싶은 작업 내용]"

새 대화든, 지금 대화를 "포크"해서 이어가든 상관없습니다 — 중요한 내용은 이 파일과 Claude의 메모리에 남아있어서, 자연스럽게 얘기해도 대부분 맥락을 다시 찾아냅니다.

## 7. Claude와 더 빠르게(적은 비용으로) 작업하는 법

1. **큰 방향은 시작할 때 한 번에 정한다.** 디자인 톤, 참고 이미지/로고, 기술 스택 같은 건 작업 시작 전에 다 던져주는 게 좋습니다. 이번 프로젝트에서 손그림 무드로 다 만든 뒤에 유니클로 스타일로 통째로 바꾼 게 제일 큰 재작업이었어요 — 로고 이미지를 처음부터 같이 줬으면 한 번에 끝났을 수 있습니다.
2. **수정 요청은 페이지별로 나누지 말고 몰아서.** "홈페이지 이 문구, about 페이지 이 문구, footer 이 문구" 이렇게 한 메시지에 다 적어서 주면, 저는 그걸 한 번에 처리합니다. 나눠서 보내면 그때마다 빌드·확인 과정이 반복돼서 시간이 더 걸려요. (뒤로 갈수록 이 방식으로 잘 해주셨어요 — 그게 훨씬 빨랐습니다.)
3. **화면 확인은 결과물만 봐도 되면 스크린샷 요청을 아낀다.** 저는 매번 코드 수정 후 실제로 브라우저 띄워서 확인하니, "진짜 반영됐는지" 걱정보다는 결과 보고를 믿고 넘어가도 대체로 괜찮습니다. 뭔가 이상해 보일 때만 캡처해서 보여주시면 됩니다.
4. **계정/로그인 관련 단계는 미리 준비해두면 좋다.** GitHub·Vercel 계정이 이미 있는지, 로그인이 되어 있는지 등을 미리 알려주면 중간에 "이거 뭐야?" 하는 왕복이 줄어듭니다.
5. **"이거 배포까지 해줘"처럼 최종 목표를 같이 말해준다.** 그러면 중간에 "배포할까요?" 되묻는 과정 없이 끝까지 처리합니다.

## 8. 이번 프로젝트에서 돌아갔던 지점 (회고)

솔직하게 정리하면:

- **디자인 전면 교체가 제일 컸어요.** 손그림 무드 → 유니클로 스타일로 갈아엎으면서 디자인 토큰, 페이지 스타일을 통째로 다시 작업했습니다. 로고 이미지가 처음부터 있었다면 이 부분은 한 번에 갈 수 있었을 거예요.
- **로그인/배포 인증 과정에서 시행착오가 있었어요.** Vercel CLI 전역 설치 권한 문제, GitHub 조직 저장소 연결 승인 등 — 이건 보안 구조상 어쩔 수 없이 사용자가 직접 해야 하는 단계라 완전히 없앨 수는 없지만, 미리 계정 상태를 알았다면 안내가 더 빨랐을 것 같습니다.
- **카피 수정이 여러 턴에 나눠졌어요.** 초반엔 footer 문구, about 문구를 따로따로 요청하셨는데, 나중에 홈페이지 문구 3곳을 한 번에 주신 방식이 훨씬 효율적이었습니다. 앞으로도 이 방식 추천드려요.
