# 프로젝트 운영 지침

이 프로젝트의 사용자는 코딩 경험이 없는 노코드 베이스 개발자다. 아래 지침을 항상 따른다.

## 가이드.md 유지관리

프로젝트 루트의 `가이드.md`는 사용자가 직접 참고하는 살아있는 문서다. 다음 상황이 생기면 사용자가 요청하지 않아도 **먼저 `가이드.md`를 갱신**한다 (git 커밋 메시지에도 반영):

- 배포/인프라 구성이 바뀌었을 때 (예: 도메인 연결, 결제 연동 추가, 새 서비스 연동)
- "이건 사용자가 직접 VS Code에서 해도 되는 작업"이 새로 생겼을 때 — 어떤 파일의 어떤 값을 바꾸면 되는지 구체적 예시와 함께 추가
- 디자인/데이터 구조 등 중요한 방향 결정이 있었을 때
- 반복될 만한 실수/막힘 지점을 해결했을 때 (다음에 또 막히지 않도록)

새 섹션을 추가할 땐 기존 섹션 번호를 다시 매기고, 문서가 너무 길어지면(대략 150줄 이상) 오래되어 더 이상 쓸모없는 내용(예: 이미 끝난 1회성 회고)은 정리하되, 링크 모음·파일 위치표·직접 하는 법 같은 상시 참조 섹션은 유지한다.

## 작업 스타일

- 새 요청이 여러 개면(문구 여러 곳, 여러 페이지 등) 한 번에 몰아서 처리한다 — 굳이 하나씩 나눠서 확인받지 않는다.
- 큰 방향 전환(디자인 톤, 기술 스택 등)이 필요해 보이면 구현 전에 먼저 확인한다 — 특히 참고 이미지/자료가 있는지 물어본다.
- 값만 바뀌는 단순 수정(가격, 재고, 문구, 매장 정보)은 "이건 VS Code에서 직접 하셔도 됩니다"라고 안내하고, 원하면 직접 처리한다.
